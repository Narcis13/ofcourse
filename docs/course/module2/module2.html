🏷️ Semantic HTML - Making Your Code Meaningful

Here's a secret: There are two ways to write HTML. The old way (using <div> for everything) and the smart way (using semantic tags). Let's learn the smart way!

💡 What does "semantic" even mean?

Semantic = meaningful. Semantic HTML uses tags that describe what the content IS, not just how it looks.

For example:

  • <header> = "This is the header"
  • <nav> = "This is navigation"
  • <article> = "This is an article"

vs. the old way: <div class="header"> = "This is... something?"

Semantic vs Non-Semantic HTML

❌ OLD WAY (Don't do this):
<div class="header">
    <div class="navigation">
        <div class="nav-item">Home</div>
    </div>
</div>

✅ SEMANTIC WAY (Do this!):
<header>
    <nav>
        <a href="#home">Home</a>
    </nav>
</header>

📝 HTML Document Structure - Building a Solid Foundation

Every HTML document follows the same basic structure. Think of it like a recipe - you need certain ingredients in the right order!

<!DOCTYPE html> ← Tells browser "this is HTML5"
<html lang="en"> ← Root element with language
<head> ← Invisible metadata
<meta charset="UTF-8">
<meta name="viewport"...>
<title>Page Title</title>
</head>
<body> ← Everything visible
<header>...</header>
<main>...</main>
<footer>...</footer>
</body>
</html>

Essential Meta Tags

<head>
    <!-- Character encoding (always use UTF-8) -->
    <meta charset="UTF-8">
    
    <!-- Mobile responsiveness -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    
    <!-- Page description for search engines -->
    <meta name="description" content="Learn web development with hands-on projects">
    
    <!-- Page title (shows in browser tab) -->
    <title>My Portfolio | Web Developer</title>
    
    <!-- Link to CSS (we'll use this in Module 3) -->
    <link rel="stylesheet" href="style.css">
</head>

🔤 Headings & Text - Organizing Your Content

Headings are like the outline of your essay - they create hierarchy and help readers (and Google!) understand your content structure.

The Heading Hierarchy

H1: Main Page Title (Use only ONCE per page!)

H2: Major Section Heading

H3: Subsection Heading

H4: Sub-subsection

H5: Rarely used
H6: Almost never used
💡 Pro Tip: Heading Best Practices
  • ✅ Use only ONE <h1> per page
  • ✅ Don't skip levels (h1 → h2 → h3, not h1 → h3)
  • ✅ Use headings for structure, not style
  • ❌ Don't use headings just to make text bigger

Text Formatting Tags

<!-- Paragraphs -->
<p>This is a paragraph. Use it for normal text blocks.</p>

<!-- Emphasis (italics) -->
<p>This is <em>emphasized</em> text.</p>

<!-- Strong (bold) -->
<p>This is <strong>important</strong> text.</p>

<!-- Line breaks -->
<p>First line<br>Second line</p>

<!-- Horizontal rule (divider) -->
<hr>

<!-- Blockquote -->
<blockquote>
    "The best time to plant a tree was 20 years ago. 
    The second best time is now."
</blockquote>

<!-- Preformatted text (preserves spacing) -->
<pre>
    This    text    keeps    its    spacing
    And line breaks too!
</pre>

🔗 Links & Navigation - Connecting the Web

Links are what make the web a "web"! They connect pages together and let users navigate your site. Let's master the art of linking!

Types of Links

<!-- External link (to another website) -->
<a href="https://github.com">Visit GitHub</a>

<!-- Internal link (to another page on your site) -->
<a href="about.html">About Me</a>

<!-- Anchor link (to section on same page) -->
<a href="#contact">Jump to Contact</a>

<!-- Email link -->
<a href="mailto:hello@example.com">Email Me</a>

<!-- Phone link (great for mobile!) -->
<a href="tel:+1234567890">Call Me</a>

<!-- Download link -->
<a href="resume.pdf" download>Download Resume</a>

<!-- Open in new tab -->
<a href="https://twitter.com" target="_blank">Twitter</a>

Building Navigation Menus

<nav>
  • Home
  • About
  • Projects
  • Contact

📋 Forms & Inputs - Collecting User Data

Forms are how users interact with your website - from login screens to contact forms. They're everywhere! Let's build some interactive forms (even though they won't actually send data yet - that comes in later modules).

Essential Form Elements

<form>
    <!-- Text input -->
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" required>
    
    <!-- Email input (with validation!) -->
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    
    <!-- Password input (hidden text) -->
    <label for="password">Password:</label>
    <input type="password" id="password" name="password">
    
    <!-- Number input -->
    <label for="age">Age:</label>
    <input type="number" id="age" name="age" min="13" max="120">
    
    <!-- Dropdown select -->
    <label for="country">Country:</label>
    <select id="country" name="country">
        <option value="">Choose...</option>
        <option value="us">United States</option>
        <option value="uk">United Kingdom</option>
        <option value="ca">Canada</option>
    </select>
    
    <!-- Radio buttons (choose one) -->
    <p>Preferred Contact:</p>
    <input type="radio" id="contact-email" name="contact" value="email">
    <label for="contact-email">Email</label>
    
    <input type="radio" id="contact-phone" name="contact" value="phone">
    <label for="contact-phone">Phone</label>
    
    <!-- Checkboxes (choose multiple) -->
    <p>Interests:</p>
    <input type="checkbox" id="web" name="interests" value="web">
    <label for="web">Web Development</label>
    
    <input type="checkbox" id="mobile" name="interests" value="mobile">
    <label for="mobile">Mobile Apps</label>
    
    <!-- Text area (multi-line text) -->
    <label for="message">Message:</label>
    <textarea id="message" name="message" rows="4" cols="50"></textarea>
    
    <!-- Submit button -->
    <button type="submit">Send Message</button>
</form>
💡 Form Best Practices
  • Always use <label> with for attribute
  • Add required for mandatory fields
  • Use appropriate input types (email, tel, number)
  • Group related inputs with <fieldset>
  • Provide helpful placeholder text
  • Consider mobile users (big touch targets!)

📊 Lists & Tables - Organizing Information

Lists and tables are perfect for organizing information in a way that's easy to scan. From navigation menus to data displays, you'll use these constantly!

Types of Lists

<!-- Unordered List (bullets) -->
<ul>
    <li>HTML</li>
    <li>CSS</li>
    <li>JavaScript</li>
</ul>

<!-- Ordered List (numbers) -->
<ol>
    <li>Learn HTML</li>
    <li>Style with CSS</li>
    <li>Add JavaScript</li>
</ol>

<!-- Description List (terms & definitions) -->
<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language - structures content</dd>
    
    <dt>CSS</dt>
    <dd>Cascading Style Sheets - styles content</dd>
</dl>

<!-- Nested Lists -->
<ul>
    <li>Frontend
        <ul>
            <li>HTML</li>
            <li>CSS</li>
            <li>JavaScript</li>
        </ul>
    </li>
    <li>Backend
        <ul>
            <li>Node.js</li>
            <li>Python</li>
        </ul>
    </li>
</ul>

Creating Tables

<table>
    <!-- Table header -->
    <thead>
        <tr>
            <th>Language</th>
            <th>Purpose</th>
            <th>Difficulty</th>
        </tr>
    </thead>
    
    <!-- Table body -->
    <tbody>
        <tr>
            <td>HTML</td>
            <td>Structure</td>
            <td>Easy</td>
        </tr>
        <tr>
            <td>CSS</td>
            <td>Styling</td>
            <td>Medium</td>
        </tr>
        <tr>
            <td>JavaScript</td>
            <td>Interactivity</td>
            <td>Hard</td>
        </tr>
    </tbody>
    
    <!-- Table footer (optional) -->
    <tfoot>
        <tr>
            <td colspan="3">Keep learning!</td>
        </tr>
    </tfoot>
</table>

🖼️ Media & Embeds - Making Pages Visual

A picture is worth a thousand words, and a video is worth a million! Let's add images, videos, and other media to make your pages come alive.

Working with Images

<!-- Basic image -->
<img src="profile.jpg" alt="My profile photo">

<!-- Image with dimensions -->
<img src="logo.png" alt="Company logo" width="200" height="100">

<!-- Responsive image -->
<img src="hero.jpg" alt="Hero image" style="width: 100%; height: auto;">

<!-- Image as a link -->
<a href="portfolio.html">
    <img src="thumbnail.jpg" alt="View my portfolio">
</a>

<!-- Figure with caption -->
<figure>
    <img src="project.png" alt="My latest project">
    <figcaption>Screenshot of my latest web app</figcaption>
</figure>
💡 Image Best Practices
  • ALWAYS include alt text for accessibility
  • Use descriptive filenames (profile-photo.jpg not IMG_1234.jpg)
  • Optimize image sizes (under 200KB for web)
  • Use appropriate formats:
    • JPEG for photos
    • PNG for graphics with transparency
    • SVG for logos and icons

Embedding Media

<!-- Video (local file) -->
<video controls width="600">
    <source src="demo.mp4" type="video/mp4">
    <source src="demo.webm" type="video/webm">
    Your browser doesn't support video.
</video>

<!-- Audio -->
<audio controls>
    <source src="podcast.mp3" type="audio/mp3">
    <source src="podcast.ogg" type="audio/ogg">
</audio>

<!-- YouTube embed -->
<iframe width="560" height="315" 
    src="https://www.youtube.com/embed/VIDEO_ID" 
    frameborder="0" allowfullscreen>
</iframe>

<!-- Google Maps embed -->
<iframe src="https://maps.google.com/maps?q=..." 
    width="600" height="450" 
    style="border:0;" allowfullscreen>
</iframe>
